## Strongest Failure Mode

The strongest realistic failure mode was a generated bootstrap that sounded complete but omitted one autonomy-critical loop: `app-wakeup-dispatch`, sent/skipped/blocked `app-wakeup-record-delivery`, `send_ready=true` gating, "direct app-thread delivery is not task completion", worker evidence, direct inbox polling, or heartbeat/no-teardown guidance.

## Disproof Attempt

The final audit used a rebuilt CLI dry-run plus explicit assertion script instead of relying on prose review. That proof checked the generated JSON fields and prompt strings emitted by `./bin/conveyor`, not only TypeScript source. Unit tests and package smoke add regression coverage so future omissions fail in CI/local smoke.
