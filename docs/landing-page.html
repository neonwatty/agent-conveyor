The database is the feedback loop

Chat is not the source of truth. Conveyor records the dispatcher state, ledger events, and proof.

manager_configs

Mode, permissions, tools, and ack policy.

app smoke receipts

Smoke ids, nonces, roles, blockers, and readiness.

routed_notifications

Messages, delivery, consumption, and failures.

acceptance_criteria

Criteria, screenshots, visual diffs, and scope calls.

autopilot policy

Heartbeat specs, applied receipts, and readiness.

telemetry_events

Heartbeats, stale work, wake actions, and clues.

When a run feels wrong

Use the database as the bug report.

Start with conveyor replay, attach the receipts, then tune the weak handoff.

Release without token wrangling

Verify the package first. Publish only through the approved Trusted Publishing run.

gh workflow run publish.yml -f version=0.1.26 -f publish=false gh workflow run publish.yml -f version=0.1.26 -f publish=true npm view agent-conveyor version
  • Verify first
    Tests, package smoke, release check, tarball.
  • Publish intentionally
    publish=true, matching version, approved environment.
  • Use OIDC
    No long-lived npm tokens.
  • Dogfood after publish
    Fresh-install before release closeout.
